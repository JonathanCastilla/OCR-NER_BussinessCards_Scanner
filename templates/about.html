About the Document Scanner

An intelligent, end-to-end pipeline for automated text extraction and semantic entity recognition.

Project Overview

This application is a custom-developed Named Entity Recognition (NER) system built to intelligently extract and categorize structured data from unstructured scanned documents.

While the underlying framework is capable of processing complex financial documents such as invoices, shipping bills, and bills of lading, this specific implementation has been optimized for processing Business Cards to ensure data privacy and accuracy. By bridging two major fields of Artificial Intelligence—Computer Vision and Natural Language Processing—the application transforms raw pixels into actionable, structured datasets.

Computer Vision

The CV module handles document detection, perspective correction (homography), spatial localization of text blocks, and Optical Character Recognition (OCR).

  • OpenCV - Boundary detection & image warping
  • PyTesseract - Optical Character Recognition
  • NumPy - Matrix and array operations
Natural Language Processing

The NLP module parses the raw OCR string, executes semantic classification via a custom-trained model, and filters the text using regex rules.

  • spaCy - Custom Named Entity Recognition
  • Pandas - Data structuring and alignment
  • Regex (re) - Lexical cleaning & parsing
System Architecture Pipeline
Step 1
Spatial Acquisition & OCR

The uploaded image is preprocessed to isolate the document. PyTesseract extracts raw text and spatial bounding box coordinates.

Step 2
Semantic Classification (NER)

The raw text is fed into a custom spaCy NER model, which was trained on BIO-tagged datasets to recognize specific entities (Names, Orgs, Phones, Emails).

Step 3
Data Alignment & Grouping

Pandas is utilized to merge the OCR spatial coordinates with the NLP semantic tags via positional indices, grouping multi-word entities together.

Step 4
Parsing & Visualization

Entities undergo strict regex parsing to remove noise (e.g., stripping letters from phone numbers). Finally, OpenCV draws labeled bounding boxes over the rectified image for the end-user.
